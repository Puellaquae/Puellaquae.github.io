需求分析

起因

这个 Repository 不单单是作为个人网站而使用，更是我前端技术的试验场地。因为我有时需要在页面中插入交互式内容或其他非文本内容，所以按照一般博客的技术流，Markdown + 模板或框架的方案并不适合于我。当然还有一个原因，如果使用工具生成网页，我就无法非常实时地在浏览器看到修改。而且我个人偏好简单纯粹的 HTML + CSS + JavaScript。但是手写 HTML 有些不那么方便而且容易出错，对纯文字内容进行程序化生成是必须的。而且现在样式紊乱，标签语义不明。本计划第二作用为尝试一次完整设计流程。

用户与定位

用户只考虑我本人。作为试验项目，我会放弃对前端框架的使用，限制对第三方库的使用（毕竟不是学习哪有）。两个主要功能：一、技术博客；二、前端技术的试验场。侧重内容的呈现。需要具有一定的扩展性。
